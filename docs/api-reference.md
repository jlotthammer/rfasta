# API Reference

The `rfasta` documentation is intentionally split into two layers:

- this MkDocs site for guides, workflows, and onboarding
- rustdoc for symbol-level Rust API reference

## Rust API Reference

The intended hosted rustdoc URL is:

- [https://jlotthammer.github.io/rfasta/api/latest/rfasta/](https://jlotthammer.github.io/rfasta/api/latest/rfasta/)

Tag-specific releases should be published under:

- `https://jlotthammer.github.io/rfasta/api/<tag>/rfasta/`

## What belongs where

Use this guide site for:

- choosing between Rust, Python, and CLI
- large-file workflow guidance
- install and setup
- conceptual behavior such as one-pass sharding semantics

Use rustdoc for:

- module-level Rust API browsing
- type and function signatures
- examples attached directly to Rust symbols
- error types and option structs
